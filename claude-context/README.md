# Контекст проекта для Claude Code

Эта папка — справка для AI-ассистента (Claude Code, Cursor и т.п.) по бэкенду
ювелирного интернет-магазина. Она объясняет: что это за проект, как он
устроен, что в нём **нельзя ломать**, что ещё нужно доделать, и как разворачивать.

## Как использовать

В начале сессии скажи ассистенту что-то вроде:

> Прочитай все файлы в `claude-context/` перед тем как что-либо менять в этом проекте. Особенно внимательно — `INVARIANTS.md`.

Дальше работайте как обычно. Если ассистент сомневается по архитектуре,
безопасности или инфраструктуре — пусть сверяется с этой папкой.

## Что внутри (в порядке чтения)

1. **`ARCHITECTURE.md`** — стек, структура папок, ключевые модули, как устроен flow данных. Прочитать первым.
2. **`INVARIANTS.md`** — **самое важное.** Архитектурные решения, которые нельзя откатить, и типичные ошибки, которые сломают auth/деплой/админку. Конкретные строки кода и почему так сделано.
3. **`BACKLOG.md`** — что ещё нужно доделать, по приоритетам. Это рабочий список задач.
4. **`DEPLOYMENT.md`** — переменные окружения, команды развёртывания БД, Docker.
5. **`HISTORY.md`** — что было сломано, как починили, какие решения и почему. Контекст, чтобы понимать, откуда взялись инварианты.

## Что НЕ в этой папке

- API-документация для фронта (см. `collection.json` — Postman, если актуальна).
- Бизнес-требования продукта (живут в голове владельца / в отдельных тикетах).
- Документация AdminJS / Sequelize / Express — гугли официальные доки.

## Стиль работы

Этот проект чинили итеративно, по одному кусту проблем за раз. Если задача
большая — разбивай её на подпункты и проходи по одному, не пытайся починить
всё разом. См. `HISTORY.md` для примеров того, как это выглядело.
